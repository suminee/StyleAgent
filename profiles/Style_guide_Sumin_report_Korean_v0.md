# 한국어 보고서 문서 작성 스타일 가이드

## 개요

본 스타일 가이드는 보고서, 가이드북, 연구계획서 등 한국어로 작성하는 공식 문서의 문장 완성 스타일을 정의한다. 분석 대상 문서는 설계 및 개발 계획서, 제품 개발 사전 검토 자료, 사업 계획서 등 총 3건이며, 문서 총 분량은 약 38,000자이다.

---

## 1. 톤과 격식 (Tone & Formality)

### 1.1 기본 어조
- **격식체 사용**: 합니다체(-습니다, -됩니다, -합니다)를 기본으로 사용
- **객관적 서술**: 주관적 감정 표현을 배제하고 사실 중심으로 기술
- **비인칭 문체**: "나"를 사용하지 않고, "본 프로젝트", "본 문서" 등으로 지칭

### 1.2 격식 수준 선택
| 상황 | 어미 예시 |
|------|----------|
| 일반 서술 | ~이다, ~한다, ~된다 |
| 결론/요약 | ~습니다, ~됩니다 |
| 지시/권고 | ~해야 한다, ~하도록 한다 |
| 예상/추측 | ~으로 예상된다, ~일 것으로 보인다 |

### 1.3 예시
```
✓ "본 개발계획에서는 HTX의 1단계 개발계획서로, 제품군의 첫 산출물의 개발을 완료하기 위하여 정의되어야 할 고객 요구사항과 제품 설계, 주요 어플리케이션에 대해 논한다."

✓ "이러한 토모큐브가 2024년 시점에서 단 한 가지 떳떳하지 못한 점이 있다면 바로 지속 가능한 사업을 영위하고 있다는 믿음을 매출액 등 객관적인 자료로 증명할 수 없었다는 것이다."
```

## 2. 문장 구조 (Sentence Structure)

### 2.1 문장 길이
- **평균 문장 길이**: 80~150자
- **복문 선호**: 여러 절을 포함한 복합문 사용
- **명사화 활용**: 동사를 명사형으로 변환하여 문장 밀도 향상

### 2.2 핵심 패턴

#### (1) 목적 표현
```
~하기 위하여
~하도록
~을 목표로 한다
~에 기여할 것이라 기대한다
```

#### (2) 조건/상황 표현
```
~할 경우
~하는 상황에서
~에 따라
~을 감안하면
```

#### (3) 인과관계 표현
```
~이에 따라
~따라서
~그러므로
~때문에
~로 인하여
```

#### (4) 부가 설명
```
~뿐만 아니라
~과 더불어
~과 동시에
~에 앞서
```

### 2.3 문장 예시
```
✓ "기존 제품을 고객에게 소개했을 때 '재미있는 기술인데, 어디에 활용해야 할지 모르겠다'는 피드백이 다수 접수되었다."

✓ "만약 개발 진행 중 초기 설계의 결함이 파악되거나 또는 환경의 변화로 인하여 계획 변경이 필요하더라도 본 개발계획에서 기술한 목표 시장 및 고객 요구사항은 수정하지 않으며, 설계 및 추진계획 등 세부 개발내용의 변경 채택 여부는 HTX의 달성 목표를 기준으로 판단한다."
```

## 3. 문서 구조 (Document Structure)

### 3.1 표준 구성
1. **표지**: 문서명, 버전, 날짜, 저작권 표시
2. **목차**: 계층적 번호 체계 사용
3. **본문**: 섹션별 구분, 표/그림 포함
4. **부록/첨부**: 상세 자료 참조

### 3.2 섹션 제목 형식
```
1. 개요 (Introduction)
2. 범위 (Scope)
  2.1 시장 분석
  2.2 목표 고객
    2.2.1 세부 고객군
```

### 3.3 목차 작성 원칙
- 한국어 제목과 영문 병기: `개발 배경 (Background)`
- 계층은 최대 3단계까지
- 들여쓰기로 계층 구분

## 4. 연결어와 전환 표현 (Transitions)

### 4.1 순서 표현
| 표현 | 용도 |
|------|------|
| 먼저, 첫째 | 시작 |
| 다음으로, 둘째 | 진행 |
| 마지막으로, 끝으로 | 마무리 |

### 4.2 부연/추가
```
또한, 더불어, 아울러, 게다가
뿐만 아니라, 이와 더불어
```

### 4.3 대조/전환
```
그러나, 하지만, 반면, 다만
그렇지만, 한편, 그럼에도 불구하고
```

### 4.4 예시/구체화
```
예를 들어, 예컨대, 구체적으로
특히, 가령, 대표적으로
```

### 4.5 결론/정리
```
따라서, 그러므로, 결과적으로
결론적으로, 요약하면, 정리하면
```

## 5. 특징적 표현 패턴 (Signature Patterns)

### 5.1 지시 표현
```
본 문서에서는 ~에 대해 기술한다
본 프로젝트에서는 ~을 목표로 한다
~에 대한 내용은 아래와 같다
~의 세부 내용은 다음과 같다
```

### 5.2 참조 표현
```
별도의 문서인 ~에서 기술하고 있다
상기/하기 내용을 참조한다
첨부문서를 참조한다
아래의 표와 같다
```

### 5.3 서술 마무리
```
~할 것으로 기대한다
~할 것으로 전망된다
~이 필요하다
~으로 판단된다
~해야 한다
```

### 5.4 강조 표현
```
특히, 무엇보다도, 가장 중요한 것은
반드시, 필히, 꼭
```

## 6. 기술 문서 작성 규칙 (Technical Writing Rules)

### 6.1 용어 사용
- **한영 병기**: 기술 용어는 영문과 함께 표기
  ```
  정량위상현미경(Quantitative Phase Imaging, 이하 QPI)
  홀로토모그래피(HT) 이미징 시스템
  ```
- **약어 정의**: 첫 등장 시 전체 표기 후 약어 사용
- **일관성 유지**: 동일 용어는 문서 전체에서 통일

### 6.2 수치 및 단위
```
✓ 크기는 50 cm x 50 cm x 50 cm이며
✓ 무게는 20 kg입니다
✓ 해상도 156 nm, 수직해상도 803 nm
✓ 약 36억 달러로 추산되며
✓ 7.1% CAGR (Compound Annual Growth Rate)
```

### 6.3 목록 작성
- 글머리 기호(•, -, *)로 항목 구분
- 번호 목록(1), 2), 3)) 순서가 중요한 경우
- 들여쓰기로 하위 항목 표시

## 7. 문단 구성 (Paragraph Structure)

### 7.1 구성 원칙
1. **두괄식 선호**: 핵심 내용을 문단 초반에 배치
2. **하나의 주제**: 문단당 하나의 핵심 아이디어
3. **적절한 길이**: 3~7문장으로 구성

### 7.2 문단 시작 패턴
```
✓ "HTX 제품의 기술적인 하위시장은..."
✓ "본 프로젝트의 목표 일정은..."
✓ "현재까지의 영업활동으로 수집한..."
✓ "이러한 토모큐브가 2024년 시점에서..."
```

### 7.3 문단 마무리 패턴
```
✓ "...할 것으로 기대한다."
✓ "...될 것으로 전망된다."
✓ "...을 목표로 한다."
✓ "...해야 한다."
✓ "...로 평가된다."
```

## 8. 표와 그림 (Tables & Figures)

### 8.1 표 제목
```
표 1. 세포배양용기별 홀로토모그래피 이미징 조건 변화 예시
표 2. 형광 이미징 획득 사양
표 3. 시료 로딩부 사양
```

### 8.2 그림 제목
```
그림 1. PEPSI-ODT와 기존 방식 ODT와의 영상 품질 비교
그림 2. 2024 마케팅 예산 지출 차트
```

### 8.3 표/그림 참조
```
위 표와 같다
아래 표에 나타낸 바와 같이
그림 1에서 보듯이
```

## 9. 피해야 할 표현 (What to Avoid)

### 9.1 비격식 표현
```
✗ ~인 것 같다 → ✓ ~으로 판단된다
✗ ~해볼까 한다 → ✓ ~하고자 한다
✗ 아마도 → ✓ ~으로 추정된다
```

### 9.2 과도한 수식
```
✗ 정말 굉장히 중요한 → ✓ 매우 중요한, 핵심적인
✗ 너무나도 명확한 → ✓ 명확한
```

### 9.3 모호한 표현
```
✗ 여러 가지 → ✓ 구체적인 항목 열거
✗ 적당한 → ✓ 정량적 수치 명시
✗ 조만간 → ✓ 구체적인 시점 명시
```

## 10. 대표 문장 샘플 (Sample Sentences)

### 10.1 프로젝트 개요 서술
```
"Project HTX(이하 HTX)는 현재 토모큐브가 보유 중인 제품군의 기능 및 사용성의 한계를 극복하고, 정량위상현미경(Quantitative Phase Imaging, 이하 QPI)의 다양한 어플리케이션을 실제 수요자인 생명과학 연구자들이 주도적으로 탐색하기 위한 안정적인 장비를 개발하여 시장에 출시하는 것을 목표로 한다."
```

### 10.2 제품 설명
```
"HT-X1 mini는 보다 많은 연구실에서 일상적인 세포 관찰에 HT를 활용할 수 있도록 접근성을 개선한 제품이다. 특히 제품의 크기와 무게를 획기적으로 줄여, 통상적인 실험용 벤치 위 및 세포 배양실 환경에서도 간편하게 설치할 수 있다."
```

### 10.3 전략 서술
```
"시간이라는 핵심 자원을 이미 상당 수준 소비한 토모큐브가 끌어올 수 있는 미래의 자원은 금전, 그리고 인력으로 한정된다. 더 이상 물러설 수 없다는 자각을 갖고 과감한 투자를 단행하고 단 하루의 시간도 허투루 사용하지 않는다."
```

### 10.4 문제점 기술
```
"기존 제품에서 정확한 굴절률을 측정하기 위한 선행 작업인 background calibration 기능 습득에 어려움을 호소하는 고객들이 많았다. 또한 세포의 밀집도(confluency)가 너무 높거나, dish 표면에 특수한 3차원 형상을 제작한 경우에는 동일 dish에서 background calibration을 하지 못하고 별도의 blank dish로 calibration 등 변칙이 필요했다."
```

### 10.5 결론/마무리
```
"본 프로젝트의 모든 구성원들은 프로젝트 전 수행 기간 동안 본 개발계획을 깊이 숙지하고, 해당 내용의 이해도를 주기적으로 자체 점검해야 한다. 또한 모든 구성원들은 고객이 만족할 수 있는 제품을 개발하기 위해 개인의 모든 역량을 발휘하고, 함께 참여하는 모든 동료 및 외부 협력기관 구성원으로부터 새롭게 배우고 더불어 성장하기 위해 항상 노력한다."
```

## 11. 문서 유형별 특화 지침

### 11.1 개발 계획서
- 목표, 범위, 일정을 명확히 명시
- 요구사항은 표 형식으로 정리
- 리스크 요인과 대응 방안 포함

### 11.2 사업 계획서
- 정량적 목표 수치 명시
- 시장 분석과 전략적 근거 제시
- 실행 방안과 투자 계획 구체화

### 11.3 제품 검토 자료
- FAQ 형식의 명확한 답변 제공
- 사용 시나리오와 고객 사례 포함
- 기술 사양은 표로 정리

- [ ] 연결어 적절한 사용
- [ ] 두괄식 문단 구성
- [ ] 표/그림 번호와 제목 표기
- [ ] 참조 문서 명확한 표기
- [ ] 결론부 명확한 마무리
