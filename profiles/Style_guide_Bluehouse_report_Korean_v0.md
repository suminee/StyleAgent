# Style Guide: 청와대 비서실 보고서 (Korean)

> 본 스타일 가이드는 2007년 대통령비서실 보고서 품질향상 연구팀에서 발간한 「청와대 비서실의 보고서 작성법」을 기반으로 작성됨

---

## 1. 메타데이터 (Metadata)

| 항목 | 내용 |
|------|------|
| 프로필명 | 청와대 비서실 보고서 스타일 |
| 언어 | 한국어 (ko) |
| 문서 유형 | 정책보고서, 상황/정보보고서, 회의보고서, 행사보고서 |
| 대상 독자 | 정책결정권자, 고위 의사결정자 |
| 핵심 가치 | 신속, 정확, 간결, 완결성 |

---

## 2. 정량적 특성 (Quantitative Metrics)

### 2.1 문장 구조
| 항목 | 기준 |
|------|------|
| 문장 길이 | 한 문장 2줄 이내 (최대 3줄 초과 금지) |
| 문단당 초점 | 한 문단에 한 개의 초점(내용)만 기술 |
| 제목 길이 | 20자 이내로 압축 |

### 2.2 서식 규격
| 항목 | 규격 |
|------|------|
| 편집 여백 | 위/아래 15mm, 좌/우 20mm |
| 머리말/꼬리말 | 10mm |
| 줄 간격 | 130% (기본) |
| 문단 간격 | 임의 설정 가능 |

### 2.3 글자체 및 크기
| 용도 | 글자체 | 크기 |
|------|--------|------|
| 본문 | 휴먼명조 | 15pt |
| 문서 제목 | 헤드라인M | 22pt |
| 중간 목차/강조 | 헤드라인M | 16pt |
| 참고 내용 (※) | 중고딕 | 13pt |
| 머리말 | 중고딕 | 14pt |
| 표 내용 | 휴먼명조 | 13~14pt |

### 2.4 항목 부호 체계
```
짧은 보고서: □ → ○ → - → ·
긴 보고서:   Ⅰ. → 1. → 1) → (1) → ①
```

**들여쓰기 규칙:**
- 첫째 항목: 문단 맨 앞
- 하위 항목: 상위 항목보다 1칸씩 오른쪽

## 3. 정성적 특성 (Qualitative Traits)

### 3.1 톤 & 격식 수준
| 특성 | 설명 |
|------|------|
| 격식 수준 | 고도의 격식체 |
| 톤 | 객관적, 전문적, 절제된 표현 |
| 감정 표현 | 미사여구/과장 표현 배제 |
| 부사 사용 | '극히', '매우' 등 부사 남용 자제 |

### 3.2 문체 특성
| 항목 | 권장 사항 |
|------|----------|
| 권장 문체 | **서술형 개조식** |
| 문장 종결 | `~하였음`, `~할 예정임`, `~함` |
| 태 선택 | 피동형보다 **능동형** 선호 |
| 어휘 선택 | 명료한 어휘, 경제적 표현 |
| 전문용어 | 설명 없이 사용 금지 (괄호로 설명 첨부) |

### 3.3 논리 전개 방식
| 특성 | 설명 |
|------|------|
| 전개 방식 | **두괄식** (핵심 먼저, 세부사항 후술) |
| 논리 체계 | 기승전결, 삼단논법, 육하원칙 준수 |
| 중요도 배열 | 가장 비중 있는 사안을 앞에 배치 |
| 결론 위치 | 도입부에 핵심 요약 제시 |

### 3.4 독자 관계
| 특성 | 설명 |
|------|------|
| 독자 설정 | 바쁜 일정의 고위 의사결정자 |
| 눈높이 조절 | 수요자의 전문지식 수준에 맞춤 |
| 추가 설명 | 불필요 (보고서 자체로 완결성 확보) |
| 행동 지침 | 수요자가 취해야 할 조치 명확히 제시 |

## 4. 문서 구조 패턴 (Structure Patterns)

### 4.1 표준 보고서 구조
```
┌─────────────────────────────────────┐
│ 머리말: 날짜, 보고서 성격/제목, 부서명   │
├─────────────────────────────────────┤
│ 제목 (제목상자 내, 연한옥색 배경)        │
├─────────────────────────────────────┤
│ 취지 (연녹색 배경, 이중테두리)           │
│ - 보고 취지나 배경 간략 서술             │
│ - 본문에 포함 시 생략 가능               │
├─────────────────────────────────────┤
│ 본문                                    │
│ □ 대항목                               │
│   ○ 중항목                             │
│     - 소항목                            │
│       · 세부항목                        │
│     ※ 참고내용 (중고딕 13pt)            │
├─────────────────────────────────────┤
│ 결론/건의사항                            │
├─────────────────────────────────────┤
│ 꼬리말: 페이지 번호                      │
└─────────────────────────────────────┘
```

### 4.2 보고서 유형별 필수 구성요소

#### 정책보고서
1. **보고 개요**: 목적, 필요성, 진행 경과
2. **현황과 문제점**: 실태, 원인분석, 대응 사례
3. **정책수단과 대안**: 정책대상, 대안, 예상 효과
4. **추진계획**: 집행계획, 홍보계획, 품질관리계획
5. **건의 및 제안**: 결정권자 조치사항 명시

#### 상황/정보 보고서
1. **제목**: 핵심사항 압축, 동작성 단어로 종결
2. **도입문**: 가장 중요한 내용 요약 (육하원칙)
3. **본문**: 중요 사안 앞 배치, 2~3줄 이내
4. **결론**: 평가, 대책, 대응방안, 조치의견

#### 회의 보고서
- **회의자료**: 회의목적, 배경, 안건 설명
- **회의결과**: 개요(목적/일시/장소/안건/참석자), 결과 요약

#### 행사 보고서
- 행사 취지, 추진 배경, 행사 시기, 전달 메시지, 기대 효과

## 5. 습관적 표현 패턴 (Signature Patterns)

### 5.1 시작 패턴
| 패턴 | 예시 |
|------|------|
| 배경 제시 | `~에 따라`, `~를 계기로` |
| 목적 제시 | `~하고자 함`, `~을 위하여` |
| 현황 기술 | `현재 ~한 상황임`, `~하고 있는 실정임` |

### 5.2 마무리 패턴
| 패턴 | 예시 |
|------|------|
| 완료 | `~하였음`, `~을 완료함` |
| 예정 | `~할 예정임`, `~할 계획임` |
| 건의 | `~을 건의드림`, `~을 검토 요망` |
| 필요성 | `~이 필요함`, `~이 요구됨` |

### 5.3 연결어/접속 표현
| 용도 | 표현 |
|------|------|
| 추가 | `또한`, `아울러`, `이에` |
| 전환 | `한편`, `다만`, `그러나` |
| 강조 | `특히`, `더구나` |
| 이유 | `~함으로`, `~인바` |
| 결과 | `이에 따라`, `그 결과` |

### 5.4 강조 표현
| 방법 | 적용 대상 |
|------|----------|
| **볼드체** | 명사 위주의 핵심 키워드 |
| 파랑색 | 중요한 부분 |
| 밑줄 | 특별 강조 시 |

**주의:** 동사, 한자체에는 볼드 사용 자제

## 6. 숫자/일시 표기법 (Number & Date Conventions)

### 6.1 날짜 표기
```
표준: 2004. 8. 15
약식: '04. 8. 15
```
- '연, 월, 일' 글자 생략, 온점(.)으로 구분

### 6.2 시간 표기
```
24시 기준, 쌍점(:)으로 구분
예: 오후 3시 20분 → 15:20
```

### 6.3 숫자 사용
- 아라비아 숫자 사용 기본
- 통계, 수치는 구체적으로 명시
- 출처 명확히 표기

---

## 7. 회피 표현 (Expressions to Avoid)

### 7.1 피해야 할 표현
| 유형 | 예시 | 대안 |
|------|------|------|
| 모호한 표현 | `~정도`, `~등` 남발 | 구체적 수치/내용 |
| 과장 표현 | `획기적인`, `대폭적인` | 객관적 서술 |
| 미사여구 | 불필요한 수식어 | 간결한 표현 |
| 피동형 | `~되어지다` | 능동형으로 변환 |
| 포괄적 제목 | `~동향`, `~현황` | 구체적 내용 반영 |

### 7.2 피해야 할 구조
- 논점과 무관한 상세 설명
- 유사 내용의 반복 서술
- 자기주장 없이 견해만 나열
- 연구논문처럼 과도하게 심층적인 내용

## 8. 품질 체크리스트 (Quality Checklist)

### 8.1 작성 전 점검
- [ ] 보고 목표가 명확한가?
- [ ] 수요자(독자)를 정확히 파악했는가?
- [ ] 최적의 보고 시점인가?
- [ ] 전체 보고서의 틀을 구상했는가?

### 8.2 내용 점검
- [ ] 보고목적에 적합한가?
- [ ] 보고내용이 정확한가?
- [ ] 간결하게 정리했는가?
- [ ] 이해하기 쉽게 썼는가?
- [ ] 완결성을 갖췄는가?
- [ ] 출처가 분명한 자료를 인용했는가?

### 8.3 형식 점검
- [ ] 표준서식을 준수했는가?
- [ ] 오탈자, 맞춤법, 시제가 정확한가?
- [ ] 기승전결의 논리체계를 갖췄는가?
- [ ] 제목이 내용을 정확히 반영하는가?
- [ ] 작성일자, 작성자 소속이 명시되었는가?

### 8.4 결론 점검
- [ ] 수요자가 취해야 할 조치가 명확한가?
- [ ] 실천 가능하고 구체적인 제안인가?
- [ ] 건의 내용이 명확히 기재되었는가?

## 9. 핵심 원칙 요약 (Core Principles)

### "칭찬받는 보고서"의 6가지 조건

1. **목적 명확** - 보고하려는 목적이 분명하게 드러남
2. **내용 정확** - 이해관계/선입견 배제, 사실 확인 철저
3. **표현 간결** - 내용과 취지가 간단명료하게 드러남
4. **이해 용이** - 추가 설명 없이도 이해 가능
5. **완결성 확보** - 추가 보고 없이 의사결정 가능
6. **적시 보고** - 최적의 시점에 보고

### 핵심 격언

> "해당 전문가 수준으로 깊이 이해하고, 비전문가도 이해할 수 있도록 쉽게 써야 한다"

> "보고서를 어렵게 쓰는 것은 쉽고, 쉽게 쓰는 것이 오히려 어렵다"

> "보고하는 사람이 이해하지 못한 내용은 보고 받는 사람도 이해할 수 없다"

> "유능한 결재권자는 1%지시, 99%감독"

## 10. 대표 예시 문장 (Sample Sentences)

### 보고 개시
```
○ ~에 따라 ~을 보고드리고자 함
○ ~의 현황 및 향후 추진계획을 다음과 같이 보고함
```

### 현황 기술
```
○ 현재 ~하고 있는 상황임
○ ~한 결과, ~로 나타남
```

### 문제점 제시
```
○ ~의 문제점이 있음
○ ~로 인해 ~한 실정임
```

### 대안 제시
```
○ ~을 위해 ~방안을 검토함
○ 1안: ~, 2안: ~ (비교표 제시)
```

### 건의/결론
```
○ ~을 건의드림
○ 상기 내용에 대해 재가를 요청드림
○ ~방향으로 추진코자 함
```
