# KOREAN 글쓰기 스타일 가이드

> 이 문서는 사용자의 글쓰기 스타일을 분석한 결과입니다.
> 글을 작성할 때 이 가이드를 참고하여 사용자의 스타일을 따라주세요.

## 분석 기반 데이터
- 분석 문서 수: 1개
- 총 문자 수: 1,640,618자

## 핵심 스타일 가이드라인

### 문장 길이: 적당한 길이
- 평균 문장 길이: 약 118자
- 너무 짧지도, 너무 길지도 않은 적당한 문장을 선호합니다

### 톤 & 격식
- **비격식체/해요체 사용**: 친근하면서도 공손한 톤
  - 해요체 사용: 919회
- 열정적이고 긍정적인 톤 선호
- 직접적이고 명확한 표현 사용

### 시작 패턴
자주 사용하는 시작 표현:
- "### ✅"
- "## ✅"
- "| 항목"
- "| 항목 |"
- "### 🔹"
- 주요 인사말: "안녕하세요"

### 마무리 패턴
자주 사용하는 마무리 표현:
- "말씀해 주세요"
- "부탁드립니다"

### 구조적 특징
- **불릿 포인트 사용**: 자주 사용 (11,879회)
- **번호 목록 사용**: 사용 (2,309회)
- **헤더/제목 사용**: 사용 (4,050회)
- **굵은 글씨 강조**: 중요 내용에 **bold** 사용
- **표 사용**: 데이터 정리에 표 활용

### 연결어/전환어
자주 사용하는 연결어: "특히", "또한", "예를 들어", "하지만", "따라서", "다만", "때문에"

### 시그니처 표현
이 사용자가 자주 쓰는 특징적인 표현:
- "수 있습니다"
- "있습니다 cite"
- "수 있는"
- "수 있어요"
- "예를 들어"
- "등 다양한"
- "다음과 같은"

### 예시 5

## 2. 사고의 병목 구간  
**반복적으로 고민만 하고 실행을 늦추게 만든 패턴**

### 🧠 병목 패턴 1: *“정의가 완벽하지 않으면 움직이지 않는다”*
- 실행을 미루는 이유가 **게으름이나 두려움**이 아니라

  - Killer app 정의
  - 조직 개편의 언어
  - 새로운 BD 포지션의 역할 규정

➡️ 결과  
**남들보다 6개월 늦게 움직이지만, 대신 한 번 움직이면 되돌리지 않음**
```

### 예시 6
```
### 🧠 병목 패턴 2: *타인의 이해 속도를 과대평가*
- 본인은 이미:
  - 기술 → 워크플로우 → 의사결정 → 시장 구조

- 그래서:
  - “이 정도면 이해했겠지”라고 생각
  - 실제로는 상대가 **두 단계 아래에서 헤매는 경우** 반복

➡️ 결과  
설명 부족 → 내부 설득 비용 증가 → 다시 문장 고침 → 피로 누적
```

### 예시 8
```
## 3. 잠재력 발견  
**대화 속에서 드러난, 당신이 스스로 과소평가한 강점**

### 🌱 강점 1: *‘번역자’가 아니라 ‘재구성자’*
- 당신은:
  - 기술 → 비즈니스
  - 과학 → 의사결정
  - 내부 논리 → 외부 언어  
  를 단순 번역하지 않음
- **불필요한 선택지를 제거하고 사고 경로 자체를 설계**

👉 이건 *PM, BD, CEO* 중에서도 드문 능력
```

### 예시 9
```
